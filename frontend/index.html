Project Overview

Built for DataPower Support Engineers

The RCA Assistant retrieves relevant sections from the official ESGA DataPower runbooks and generates grounded, cited recommendations. Every answer traces back to a source — no hallucination.

🔍

Hybrid Retrieval

Combines dense vector search (pgvector), sparse full-text search (Supabase FTS), and error-code metadata filtering — merged with Reciprocal Rank Fusion for best recall across all query types.

📌

Grounded Answers

Every factual claim is cited as [Pattern N — Name | Section]. The model is strictly prohibited from using prior knowledge or speculating beyond the retrieved context.

🔗

Multi-turn Conversation

Maintains a 10-turn history window with automatic query reformulation — follow-up questions like "what about DNS?" are resolved against prior turns before retrieval.

🚨

Smart Escalation Routing

Detects escalation queries and pins Escalation Matrix chunks to the top retrieval slots, ensuring the right contact group, condition, and SLA are always surfaced first.

🛡️

Scope Guardrails

Queries outside the runbook scope (configuration, setup, HA design) receive a fixed refusal rather than a hallucinated answer. Out-of-scope detection is enforced at the system prompt level.

📊

Built-in Evaluation

57-case golden test set with RAGAS metrics (faithfulness, answer correctness, relevancy) plus custom retrieval metrics (HR@5, MRR, citation accuracy, escalation routing accuracy).

Project Setup

Getting Started

Prerequisites: Python 3.11+, a Supabase project with pgvector enabled (supabase/schema.sql), and an OpenAI API key.

1

Clone and install

git clone <repo-url>
cd RootCauseAnalyzer-Chatbot
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS / Linux
pip install -r requirements.txt
2

Configure environment

Copy .env.example to .env and fill in your credentials:

SUPABASE_URL=https://<project-id>.supabase.co
SUPABASE_SERVICE_KEY=<service-role-key>
OPENAI_API_KEY=<your-openai-api-key>
3

Ingest runbooks

Parses the runbooks in source/, creates embeddings, and loads them into Supabase. Run once, or whenever runbook content changes.

python ingest.py
4

Start the server

python -m uvicorn api.index:app --host 0.0.0.0 --port 8000

Open http://localhost:8000 in your browser.

5

Vercel deployment (optional)

The project includes a vercel.json for serverless deployment. Set SUPABASE_URL, SUPABASE_SERVICE_KEY, and OPENAI_API_KEY as Vercel environment variables. The ingestion script is local-only and must not be deployed.
